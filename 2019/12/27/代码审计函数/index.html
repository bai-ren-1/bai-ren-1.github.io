代码审计函数

发表于 2019-12-27 | 分类于 WEB安全

代码审计函数

SQL注入

SQL较多发生于下面这些功能点上：

  • 登录页面
  • 获取HTTP头（user-agent/client-ip)
  • 订单处理（常发生二次注入）

二次注入是指已存储（数据库、文件）的用户输入被读取后再次进入到 SQL 查询语句中导致的注入 。

即处理这个输入时进行了转义，但是存储进数据库中又变成了脏数据，然后使用到这个脏数据时就会发生注入。

普通注入关注函数：

  • select from
  • mysql_connect
  • mysql_query
  • mysql_fetch_row
  • update
  • insert
  • delete

编码注入关注函数：

  • urldecode/rawurldecode
  • mysql_set_charset(‘GBK’)

防范方法：

  • gpc/runtime魔术引号 （PHP配置）
  • 转义类函数
    • addslashes对单双引号、反斜杠和空字符进行转义
    • mysq_ [ real_ ] escape_string同理对指定字符串进行转义
  • intval等字符转换（适用于int型注入）
  • PDO预编译

SSRF

SSRF关注函数：

  • curl
  • file_get_content
  • fsockopen

XSS

XSS漏洞关键在于：寻找没有被过滤的参数和输出这些参数的输出函数

XSS关注函数：

  • print
  • print_r
  • echo
  • printf
  • sprinf
  • sprintf
  • die
  • var_dump
  • var_export

防范：

  • 对特殊字符 ‘ “ < > \ : and & # 在输出和二次调用时进行HTML实体转义
  • 黑白名单方式拦截标签事件属性（自定义正则规则）

CSRF

CSRF主要是用于越权，常发生于控制权限的地方：

  • 管理后台
  • 会员中心
  • 论坛帖子
  • 交易管理

关注点：

  • 黑盒方法：

    • 抓包看有没有token
    • 去掉refer再重发一次看响应是否一样

  • 白盒方法：

    • 查看引用的核心基础文件有没有token关键字或者验证操作

防范方法：

  • token
  • 验证码

文件包含

文件包含关注的地方：模块加载，cache调用的地方

文件包含关注函数：

  • include
  • include_once
  • require
  • require_once

once ： 只包含一次

require ： 包含并执行，报错就退出程序

include ： 包含并执行，报错也继续执行下面的代码

文件上传

文件上传关注函数：

  • move_uploaded_file

代码执行

关注函数：

  • eval
  • assert
  • call_user_func_array
  • preg_replace
  • call_user_func
  • 动态函数 $a($b)

命令执行

关注函数：

  • system
  • exec
  • passthru
  • shell_exec
  • popen
  • proc_open
  • pcntl_exec

防范方法：

用白名单对用户输入进行过滤

  • 使用自定义函数或函数库来替代外部命令的功能
  • 使用 escapeshellarg()函数来处理命令参数
  • 使用 safe_mode_exec_dir 指定可执行文件的路径

变量覆盖

变量覆盖关注函数：

  • extract
  • parse_str
  • import_request_variables
